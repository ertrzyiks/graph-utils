Type

Graph

Graph type is a core of the library. It accepts two generic parameters: shape of the custom properties of the nodes and edges.

Graph<NodeData = {}, EdgeData = {}>
Type

NodeId

The library supports only string ids for the nodes. To distinguish it from other string values all functions taking node is as a param use NodeId alias to hint it's purpose.

NodeId = string

createGraph

Initialize an empty graph object without nodes and edges. The two generic parameters allows to constraint the type of custom properties of nodes and edges. Those constraints are respected by other functions.

createGraph(): Graph

Return value

GraphA new instance of a graph

Example 1

const graph = createGraph()

Example 2

type NodeData = { label: string }
  const graph = createGraph<NodeData>()

Example 3

type EdgeData = { weight: number }
  const graph = createGraph<{}, EdgeData>()

Example 4

type NodeData = { label: string }
  type EdgeData = { weight: number }
  const graph = createGraph<NodeData, EdgeData>()
Function

findClosestPaths

Traverse graph and precalculate the closest paths between the source node (starting point) and all other nodes. Use `retrieveClosestPath` and `retrieveClosestPathDistance` to get the details out of the results.

findClosestPaths(graph, params): ClosestPathResults

Parameters

NameTypeDescription
graphGraph
paramsParams
from
NodeIdThe source node
getDistance
(edge: EdgeData) => number

Return value

ClosestPathResults

Example 1

const results = findClosestPaths(graph, {
    from: '1',
    getDistance: edge => edge.weight
  })
  const ids = retrieveClosestPath(results, { to: '2' })
